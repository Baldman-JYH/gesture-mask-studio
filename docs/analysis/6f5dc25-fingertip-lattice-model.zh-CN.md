## 逐帧对比发现

参考效果更接近“手部驱动的折叠空间模板”，而不是半透明的屏幕空间覆盖层。它显示的是一个动态多面对象，双手移动时可见轮廓持续变化：有些帧像长条，有些帧像宽面板，有些帧像折叠或弯曲的带状面。不同面具有不同视觉处理：实时画面采样、彩色半透明面板、边缘高光、点阵/图形面、较暗的侧面或背面。

当前实现仍然更像“每只手一个锚点驱动的固定空间模板”。它已经可以显示透视和多个材质组，但拓扑结构不是由指尖生成的。实际表现仍然是一个宽的折叠片/带状体跟随手移动，而不是由十个指尖控制点构建出的几何体。

这与当前代码边界一致：

- `deriveGestureAnchorFrame` 把每只手压缩为“拇指指尖和食指指尖的中点”。
- `buildSpatialTemplateMesh` 基于这些锚点生成固定的单手或双手模板。
- renderer 可以绘制 mesh 面，但 model 层尚未暴露指尖拓扑。

## 对用户方案的判断

“点-线-面-体”的方向是正确的，并且比当前锚点模板方案更接近参考视频。把指尖作为语义控制点，是更合理的领域模型；后续变更样式时，可以通过替换拓扑规则和材质规则扩展，而不是重写摄像头、追踪和渲染流程。

但原始方案需要增加数学约束后才能实现。仅靠十个指尖点，不能保证在所有手势下都形成稳定闭合的三维体。

## 数学风险

1. **单手指尖闭环不一定共面。** 单手 `A-B-C-D-E-A` 在手指弯曲、重叠、朝向摄像头旋转时，可能变成非共面、凹多边形或自相交多边形。因此不能不经验证就把它当成一个面。

2. **十个指尖首先定义的是图或曲面，不是闭合体。** 一个完整几何体需要有向面、法线方向和厚度。指尖可以定义可变形网格；实现时必须沿稳定法线偏移前表面来生成厚度。

3. **退化姿态很常见。** 如果指尖共线、距离过近、被遮挡或交叉，部分四边形会坍缩或翻面。mesh builder 必须跳过无效面，或降级到更小拓扑。

4. **左右手和镜像必须归一化。** camera-space 左右、MediaPipe handedness、display-space 镜像状态可能互相不一致。模型必须在明确坐标转换后，基于 display-space 构建拓扑。

5. **深度值有噪声且不是公制。** MediaPipe hand `z` 适合表达相对远近，但不是精确世界坐标。深度应该经过平滑和夹取后影响折叠幅度和透视，而不能直接当成真实厚度。

6. **拇指的解剖顺序和其他手指不同。** 拇指可能明显偏离四指弧线。`E-A` 闭合边默认不应该强制变成面，除非通过面积、共面和相交检查。

7. **追踪丢失需要回退。** 真实摄像头帧可能丢失一只手或某个指尖。拓扑层必须支持置信度阈值和回退 mesh，不能直接生成破损几何体。

## 推荐数学模型

使用 `FingertipLatticeFrame` 作为主模型。

语义控制点：

- `A`：拇指指尖，landmark 4。
- `B`：食指指尖，landmark 8。
- `C`：中指指尖，landmark 12。
- `D`：无名指指尖，landmark 16。
- `E`：小指指尖，landmark 20。

稳定辅助点：

- 手腕：landmark 0；
- 掌指关节：landmarks 5、9、13、17；
- 由手腕和掌指关节推导掌心中心与掌面法线；
- 每个点需要置信度，并结合上一帧状态做时间平滑。

当两只手有效时，构建主网格：

- 横向连线：`L_A-R_A`、`L_B-R_B`、`L_C-R_C`、`L_D-R_D`、`L_E-R_E`。
- 侧边连线：`L_A-L_B-L_C-L_D-L_E` 和 `R_A-R_B-R_C-R_D-R_E`。
- 主面条带：
  - `Q_AB = [L_A, R_A, R_B, L_B]`
  - `Q_BC = [L_B, R_B, R_C, L_C]`
  - `Q_CD = [L_C, R_C, R_D, L_D]`
  - `Q_DE = [L_D, R_D, R_E, L_E]`

不要默认强制把 `E-A` 闭合成面。它可以作为边线、可选封口，或在通过面积、共面和相交检查后作为特定样式面。

每个四边形在渲染前必须三角化。对于非共面四边形，选择扭曲更小且不翻转法线的对角线。随后沿平滑法线复制前表面，连接侧面，生成厚度。每条带、封口、背面和边缘都应分配稳定的 material id。

## 实现可行性

当前技术栈仍然可行。Browser + MediaPipe Hands + Three.js 可以实现这类效果。问题不在技术选型，而在当前模型边界。

需要的模块调整：

- 新增 `features/hand-topology/`，把 MediaPipe landmarks 映射为语义指尖和稳定辅助点。
- 新增 `features/fingertip-lattice/`，负责连线、有效面、三角化、法线、厚度和材质组。
- 把 `features/spatial-template-model/` 从固定锚点模板改为拓扑网格驱动的 mesh。
- `features/spatial-template-renderer/` 基本保留为 mesh renderer，但要确保支持 triangle list 和 material groups。
- 当前锚点模板路径只应保留为 fallback/debug 模型，不应继续作为主目标行为。

## 开发前测试要求

实现前先补 RED 测试：

- MediaPipe landmarks 在 display-space 镜像归一化后，能正确映射为左右手 `A/B/C/D/E` 指尖语义。
- 两只有效手能生成 5 条横向连线和 4 个主面条带。
- 非共面四边形能被三角化为 winding 稳定的有效三角形。
- 退化或缺失指尖会跳过无效面，而不是生成翻面三角形。
- `scene`、`panel`、`back`、`accent`、`edge` 等 material id 稳定。
- 同一只物理手被重复识别时，不会生成双手网格。
- 单手回退使用受控的局部拓扑网格，而不是旧的固定三角形/楔形。

## 结论

用户提出的“点-线-面-体”方案可行，但应实现为“带校验的指尖拓扑网格”：以指尖为主控制点，以掌部关键点稳定法线和厚度，通过三角化、退化处理、材质分组和回退状态生成最终 mesh。这个方向应取代当前固定双锚点空间模板，作为长期主模型。
