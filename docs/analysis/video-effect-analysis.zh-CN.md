## 视觉行为观察

### 术语修正

`Mask` 适合作为项目名中的简称，但参考视频中的效果更准确地说是“手势驱动实时采样光片”。它不是简单遮挡层，也不是只覆盖人脸。光片会重新渲染其背后的实时摄像头内容，并叠加纹理、边缘线、色调、透明度和高光。

### 主效果

效果是在摄像头画面前绘制一张半透明的实时采样图像光片。光片由用户双手锚定，表现得像一张薄而可变形的平面：

- 在左右手之间拉伸。
- 随手部位置变化旋转和倾斜。
- 可呈现长四边形条、梯形或三角楔形。
- 带有明亮白色边线和光泽高光，使其看起来像实体光片，而不是平面贴纸。
- 光片内部包含来自其背后区域的实时摄像头信息；人脸、手、衣服和背景结构都可能按预设样式被重新渲染。

## 手势和形状时间线

原始算法无法从视频中完全确认，但可观察到以下状态：

- 约 0s：光片是较大的三角/四边形楔形，上部有绿色纹理，下部有扑克牌纹理。
- 约 1s：光片变成横跨脸部的蓝色技术线稿四边形。
- 约 2s：光片变成双手之间的细长扑克牌条。
- 约 3s：光片在保持手部对齐几何的同时逐渐切换到绿色插画条。
- 约 4s：左手靠近摄像头并形成三角形手势区域，光片收缩成竖向三角楔形。
- 约 5s-7s：光片重新展开为绿色三角/梯形，左手在前景较大，右手靠近远端边缘。
- 约 8s-9s：光片展开成大面积蓝色技术线稿梯形，几乎占据中间和右侧区域。

## 纹理状态

至少出现三类纹理：

1. 蓝色技术线稿
   - 蓝/青色渐变背景。
   - 白色线稿和图纸式笔画。
   - 呈现明显未来科技感。

2. 白色扑克牌图案
   - 白色底。
   - 红色牌面符号。
   - 看起来像较薄的实体卡片条。

3. 绿色插画/自然纹理
   - 绿色和黑色有机线条。
   - 比扑克牌状态更暗、更不透明。

视频不能证明纹理切换是由明确手势、手距阈值、定时轮换还是隐藏 UI 触发。新项目首版推荐采用确定性控件切换，后续再扩展自动手势切换。

## 手势推断

最可能的控制模型：

- 检测双手。
- 使用手指关键点作为锚点，尤其是食指尖和拇指尖。
- 从锚点生成 2D 多边形。
- 将实时摄像头采样和样式纹理映射到多边形内，并做透视变形。
- 对关键点运动做逐帧平滑，减少抖动。

最强实现线索是光片端点紧贴手部/指尖位置，而内部显示的是背后内容的风格化重渲染。这更符合“手部关键点 + WebGL 视频采样合成”，而不是逐帧图像生成或静态贴图覆盖。

## 实现影响

### 实时识别

必须在浏览器端实时处理摄像头帧。核心依赖：

- `getUserMedia` 获取摄像头流。
- MediaPipe Tasks Vision `HandLandmarker` 获取手部关键点。
- WebGL/Three.js 将视频帧作为纹理采样。

### 光片渲染

光片内部不是直接遮盖，而是：

1. 使用当前视频帧作为 `VideoTexture`。
2. 将光片顶点映射到屏幕空间。
3. 根据光片覆盖区域采样实时视频。
4. 在 shader 中叠加预设纹理、色调、高光和边缘线。
5. 输出半透明合成结果。

### 首版可行目标

首版无需完全复刻原视频算法，但必须满足：

- 双手控制光片形状；
- 光片内部实时采样摄像头；
- 至少 3 个视觉预设；
- 样式切换可重复、可解释；
- 在 GitHub Pages 静态部署中可运行。

## 验收重点

- 移动手或背景物体时，光片内部内容必须同步变化。
- 光片不应只是蓝色/绿色/扑克牌图片覆盖。
- 人脸只是采样内容的一种示例，衣服、手、背景同样应该被采样。
- 双手移动、旋转、拉伸时，光片几何必须跟随变化。
