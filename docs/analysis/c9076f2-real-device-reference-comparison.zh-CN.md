## 最新测试发现

上一轮上下颠倒问题已经修正。当前光片内部采样到的摄像头内容不再上下倒置。

剩余可见问题是稳定的采样偏移：光片内部内容无法与可见摄像头预览中的同一内容完全重合。根因是可见 `<video>` 使用了 `object-fit: cover`，浏览器在显示摄像头流时会先裁剪和缩放；而 renderer 之前直接按完整视频纹理采样，没有应用同一套 cover 裁剪映射。

对于当前二维实现，单手出现光片应视为 bug。早期原型中存在 `one-hand-preview` 模式，但当前可部署的二维光片应只在确认两只手时渲染；零只或一只手都应输出隐藏几何。

## 参考视频发现

重新获取的参考视频不是扁平的摄像头采样矩形，而是手指锚定的三维模板：

- 多面材质状态；
- 透视缩放和近大远小；
- 围绕手部/指尖锚点折叠和旋转；
- 模板不同面承载不同视觉设计；
- 由指尖驱动移动，而不只是两只手掌中心之间的屏幕空间矩形。

当前 renderer 仍可用于验证实时场景采样，但它的架构不足以完整复现参考视频。对齐参考效果的主线仍然是 ADR-0002：手势锚定三维模板模型。

## 修复方向

本次修复保持在当前二维实现范围内：

- 移除当前运行时的单手预览渲染；
- 保留双手光片渲染；
- 视频 UV 采样前先把 display-space 点通过可见 `object-fit: cover` 变换反算到源视频坐标；
- 水平镜像和垂直纹理坐标保持独立规则；
- 完整三维模板模型留到 ADR-0002 实施阶段。

## 验证要求

自动化：

- 运行手势状态、几何、场景采样和 renderer UV 目标测试；
- 完整 `npm test`；
- `npm run build`；
- 中英文文档配对检查；
- `git diff --check`。

真实设备：

- 只有一只手时：摄像头区域不应渲染光片；
- 两只手可见时：光片应渲染；
- 光片内部采样内容应比 `c9076f2` 更接近可见背景位置；
- Mirror 开/关都应保持水平对齐；
- 控制台不应出现 `THREE.WebGLProgram: Shader Error`。
