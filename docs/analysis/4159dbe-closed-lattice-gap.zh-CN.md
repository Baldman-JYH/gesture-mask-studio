## 发现

`4159dbe` 相比更早的固定锚点模型有进步，因为渲染几何更稳定地跟随指尖拓扑。但是它仍未完整实现用户要求的点-线-面-体模型。

可见差距：

- 单手模式仍渲染为条带/ribbon，因为它创建了虚拟第二只手。它应改为由 `A-B-C-D-E-A` 围成的单手面。
- 双手模式仍是开放的。它创建了 `AB`、`BC`、`CD`、`DE` 条带，但没有创建 `EA` 闭合条带。
- 双手模式缺少左右手各自 `A-B-C-D-E-A` 闭环形成的端面。
- 面材质过于全局化。大多数可见颜色变化由 active preset tint 驱动，因此对象经常像一整张面同步变色，而不是稳定显示不同面不同颜色。
- 在靠近或交叉手势下，由于缺少闭合结构，对象视觉上容易坍缩成长的开放板。

## 根因

当前实现仍保留第一版指尖拓扑网格的折中设计：

- 单手 -> 创建虚拟 rail；
- 双手 -> 创建四个开放条带；
- 端面 -> 缺失；
- `EA` 闭合 -> 缺失；
- material id -> 按粗粒度条带分组，而不是完整拓扑角色。

这意味着当前领域模型仍是可变形条带网格，不是完整的点-线-面-体几何体。

## 必要模型变更

下一轮实现应使用以下拓扑：

单手：

- 点：`A`、`B`、`C`、`D`、`E`；
- 边：`AB`、`BC`、`CD`、`DE`、`EA`；
- 面：由五点闭环三角化形成一个前表面；
- 可以为了可见性增加厚度、背面和边缘面，但不能使用虚拟第二只手。

双手：

- 点：`L_A` 到 `L_E`，以及 `R_A` 到 `R_E`；
- 横向边：`L_A-R_A`、`L_B-R_B`、`L_C-R_C`、`L_D-R_D`、`L_E-R_E`；
- 侧边闭环：`L_A-L_B-L_C-L_D-L_E-L_A` 和 `R_A-R_B-R_C-R_D-R_E-R_A`；
- 条带面：`AB`、`BC`、`CD`、`DE`、`EA`；
- 端面：左手 `A-B-C-D-E-A` 和右手 `A-B-C-D-E-A`；
- material id 按拓扑角色分配，保证相邻面视觉上有明确区分。

## 验证含义

自动化测试必须断言：

- 单手模式不使用虚拟 rail；
- 单手模式具有 `AB/BC/CD/DE/EA` 五条边；
- 双手模式包含五个条带，包括 `EA`；
- 双手模式包含左端面和右端面；
- material id 包含 scene、panel、cap、accent、back、edge 等不同角色；
- 所有 renderer face 都是非零面积三角形。

真实设备验证需要确认：

- 单手产生紧凑的手部闭环面，而不是长条虚拟带；
- 双手产生带明显端面的闭合体；
- 相邻面在移动时仍保留不同颜色/材质处理；
- 交叉或靠近姿态不再坍缩为单张开放长板。
